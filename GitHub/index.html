GitHub — Introducción general

Plataforma de desarrollo colaborativo basada en Git. Permite alojar repositorios, revisar código, gestionar issues y releases, y trabajar en equipo con forks y pull requests.

En esta presentación repasamos comandos básicos y conceptos útiles para el día a día.

Comandos básicos — Enviar cambios

Enviar cambios al remoto

git push -u origin main // Primera vez: asociar rama local con remota
git push // Enviar cambios (después de la primera vez)
git push --tags // Enviar tags al remoto
Recomendación: asociar tags con releases en GitHub para documentar versiones.

Comandos básicos — Traer y clonar

Traer actualizaciones y clonar

git pull // Traerse las últimas actualizaciones del remoto
git clone https://github.com/X.git // Clonar un repositorio
git config pull.rebase true // Configurar localmente el rebase al hacer pull

Conceptos de GitHub

Fork — Copia de un repositorio público en tu cuenta. Te permite proponer cambios al proyecto original sin tener acceso de escritura directo.

Pull request (PR) — Solicitud para enviar tus contribuciones (cambios de tu fork o rama) al repositorio central. El mantenedor puede revisar y fusionar.

Conceptos de GitHub (cont.)

Actualizar fork — Sincronizar tu fork con el repositorio central: traerte los últimos cambios del proyecto original a tu copia (por ejemplo con git pull upstream main si tienes el remoto upstream configurado).

Gist — Minirepositorio público para compartir fragmentos de código, notas o archivos pequeños sin crear un repo completo.

Gist y Visual Studio Code

Para usar Gists desde VSC necesitas un GitHub Personal Access Token (PAT) con permiso gist.

  • Seleccionar perfil: Cmd+Shift+P → GIST: Select Profile → pegar tu GitHub PAT.
  • Abrir tus Gists: Cmd+Shift+P → GIST: Open Gist → se muestra tu lista de Gists.

Así puedes crear, editar y abrir Gists directamente desde el editor.
